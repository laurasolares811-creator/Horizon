## [Bun Rewritten from Zig to Rust with Agentic Engineering](https://simonwillison.net/2026/Jul/8/rewriting-bun-in-rust/#atom-everything) ⭐️ 8.0/10

Bun's creator Jarred Sumner detailed the process of rewriting the Bun runtime from Zig to Rust, leveraging AI coding agents, a TypeScript conformance suite, and adversarial review to complete the port in 11 days. This demonstrates that AI coding agents can now enable ground-up rewrites of major software projects, potentially reducing memory-related bugs and reshaping language choice decisions in systems engineering. The rewrite cost an estimated $165,000 in API tokens, consuming 5.9 billion input tokens, and the Rust version has been live in Claude Code since June 17th, with 10% faster Linux startup.

rss · Simon Willison · Jul 8, 23:57

**Background**: Bun is an all-in-one JavaScript runtime originally written in Zig, a systems programming language with manual memory management. Mixing garbage collection with manual memory in Zig led to frequent use-after-free and double-free bugs. Rust's ownership model provides compile-time memory safety, making such errors impossible in safe code. The rewrite used Bun's existing TypeScript test suite as a conformance suite to validate correctness automatically.
