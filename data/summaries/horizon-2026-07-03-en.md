## [Preprint Claims Markets Are Competitive if and Only if P ≠ NP](https://arxiv.org/abs/2602.20415) ⭐️ 7.0/10

A new preprint by Philip Maymin proves mathematically that markets are competitive if and only if P ≠ NP, implying that when P = NP, markets become collusive. As AI enhances firms' computational power, the paper suggests markets may shift from competitive to collusive regimes without explicit coordination. This work provides a theoretical foundation linking computational complexity to market behavior, with significant policy implications. It suggests that advancing AI could undermine competition without explicit collusion, challenging antitrust enforcement and regulation of algorithmic pricing. The paper is an unverified preprint and its title actually reads 'Markets are competitive if and only if P != NP,' but Hacker News erroneously displayed 'P = NP.' The argument hinges on the widely believed but unproven conjecture that P ≠ NP. This preprint extends the author's 2010 paper, which claimed markets are efficient if and only if P = NP.

hackernews · kscarlet · Jul 3, 15:41 · [Discussion](https://news.ycombinator.com/item?id=48776345)

**Background**: P vs NP is a fundamental problem in computer science: P problems can be solved efficiently (polynomial time), while NP problems have solutions that can be verified efficiently but are believed hard to solve. If P = NP, many intractable problems would become easy. Algorithmic collusion occurs when pricing algorithms independently learn to sustain supra-competitive prices without explicit communication. Economist Friedrich Hayek argued markets use dispersed knowledge that no central planner can fully capture. This paper formalizes such ideas computationally.

## [crustc: Entirety of rustc Translated to C](https://github.com/FractalFir/crustc) ⭐️ 7.0/10

crustc is a Rust-to-C transpiler that translates the entire rustc compiler (version 1.98.0-nightly) into C code, allowing it to be compiled with GCC for platforms without LLVM support. This enables Rust compilation on legacy or obscure hardware without LLVM or GCC, opens new bootstrapping paths, and supports security auditing through techniques like Diverse Double-Compiling. It is a direct translation of rustc 1.98.0-nightly, preserving exact semantics, and compiles to C using GCC. The project is ongoing, with the compiler implementation to be released later.

hackernews · Philpax · Jul 2, 22:57 · [Discussion](https://news.ycombinator.com/item?id=48768464)

**Background**: Rust's official compiler, rustc, uses LLVM as a backend to generate machine code. For platforms without LLVM, alternative backends are needed. Compiler bootstrapping is the process of building a compiler for a language using a compiler written in that same language, often requiring a pre-existing compiler. Diverse Double-Compiling (DDC) is a method to verify compiler trustworthiness by using two different compilers to compile the same source and comparing the outputs bit-for-bit.
